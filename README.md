# Dwarven-Smart-GIT-Sync

**Safe, multi-repo, multi-account Git sync tool. Part of the [DwarvenSuite](https://github.com/gitdwarf).**

Designed for unattended automation *and* paranoid humans. Keeps your local repos in sync with GitHub -- in both directions -- without ever surprising you.

## Philosophy

- **No surprises** -- every decision is logged before it happens
- **Explicit authority** -- nothing is committed, pushed, or deleted without your say-so
- **Repairable by humans at 2am** -- every failure prints exactly what went wrong and what to run to fix it

## What it does

Point it at a directory of cloned repos and run it. It fetches, works out which direction each repo needs to go, and acts:

- **Local ahead of remote** -- pushes
- **Remote ahead of local** -- pulls
- **Already in sync** -- moves on silently
- **Both sides moved** -- stops, prints the exact git commands to resolve it manually, and leaves everything untouched

Supports multiple GitHub accounts, SSH keys and Personal Access Tokens, encrypted credential storage, and a drop-in XML config file for teams.

On first run, a setup wizard walks you through everything.

## Features

- **Two-way sync** -- push and pull, decided per-repo on each run
- **Multi-account** -- separate GitHub accounts in separate subdirectories, each with its own credentials
- **Encrypted credential store** -- hardware-fingerprinted (machine-id + root UUID + USER). A stolen credential file is useless on another machine
- **SSH key and PAT support** -- auto-detected; SSH key passphrase detection with actionable warnings
- **XML config file** -- fully generated by the script; drop in a template for team/corp use with credentials left blank
- **Branch awareness** -- detached HEAD and untracked branches skipped cleanly; non-default branches warned before touching
- **Dirty repo handling** -- three explicit modes: skip (default), warn-and-proceed (`allow_dirty`), or stash-pull-pop (`auto_stash`)
- **Auto credential rotation** -- if a sync fails with an auth error, re-prompts interactively and saves the new credential
- **Non-interactive safe** -- cron and automation contexts detected via tty; never hangs waiting for input
- **`--dry-run`** -- shows exactly what would happen, direction per repo, without touching anything
- **`--clone-missing`** -- fetches your GitHub repo list via API and clones anything not present locally
- **`--create-repo`** -- creates a new GitHub repo and clones it locally, with confirmation
- **Email log delivery** -- pipes the run log through the system MTA after each run (optional)
- **`--status`** -- shows mode, accounts, auth methods, partial token display, repo counts
- **`--list-repos`** -- filesystem scan showing repos that would be synced and their current branch; zero network, zero git operations

## Installation

    git clone https://github.com/gitdwarf/smart-git-sync.git
    cd smart-git-sync
    chmod +x smart-git-sync.sh

Optionally, symlink it onto your PATH:

    ln -s "$PWD/smart-git-sync.sh" ~/.local/bin/smart-git-sync

No dependencies beyond bash 4.0+, git, and curl.

## First run

    smart-git-sync.sh

With no accounts configured, the setup wizard runs automatically. It will ask for:

- Your GitHub repos root directory (e.g. `~/GitHub`)
- A config file location (recommended -- enables drop-in team templates)
- A credential store location (encrypted; leave blank to always prompt)
- Your first GitHub account and credentials (SSH key path or PAT)

After setup, clone your repos into the configured directory and run again to sync.

## Directory layout

**Single account:**

    ~/GitHub/
      repo-one/
      repo-two/
      repo-three/

**Multiple accounts:**

    ~/GitHub/
      personal/
        repo-one/
        repo-two/
      work-bot/
        corp-repo/

## Usage

    # Sync all repos
    smart-git-sync.sh

    # See what would happen without doing it
    smart-git-sync.sh --dry-run

    # Show configured accounts and repo counts
    smart-git-sync.sh --status

    # List repos that would be synced (branch, skip reason if any)
    smart-git-sync.sh --list-repos

    # Clone repos on GitHub not yet present locally
    smart-git-sync.sh --clone-missing

    # Create a new GitHub repo and clone it locally
    smart-git-sync.sh --create-repo my-new-tool

    # Create a repo under a specific account (multi-account)
    smart-git-sync.sh --create-repo my-new-tool --account work-bot --description "Internal tool"

    # Add or remove an account
    smart-git-sync.sh --add-account
    smart-git-sync.sh --remove-account --account old-bot

    # Reset config and credential files (accounts must be removed first)
    smart-git-sync.sh --reset

    # Print version
    smart-git-sync.sh --version

## Config file

The config file is XML, generated and managed by the script. Drop a template into place for team use -- leave credentials blank and each user fills in their own via the script.

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!--
    smart-git-sync configuration
    This file is managed by smart-git-sync.
    Invalid or unrecognised values are ignored and will be overwritten on next save.
    Do not add credentials here -- those are stored separately in password_store.
-->
<smart-git-sync>
    <github_root>/home/user/GitHub</github_root>
    <accounts>
        <account>personal</account>
        <account>work-bot</account>
    </accounts>
    <safety>
        <allow_dirty>false</allow_dirty>
        <fast_forward_only>true</fast_forward_only>
        <auto_stash>false</auto_stash>
    </safety>
    <logging>
        <enabled>true</enabled>
        <log_dir>/var/log/smart-git-sync</log_dir>
        <email_logs>false</email_logs>
        <log_email_address></log_email_address>
    </logging>
    <password_store>/home/user/.config/smart-git-sync/.credentials</password_store>
</smart-git-sync>
```

Set `cfg_path` in the script's user variables section to point at your config file.

## Dirty repo options

Three mutually exclusive modes -- set in config or user variables:

| Setting | Behaviour |
|---|---|
| `allow_dirty=false` (default) | Skip dirty repos with a warning |
| `allow_dirty=true` | Warn but sync anyway |
| `auto_stash=true` | Stash changes, sync, pop. If pop fails, stash is preserved and the exact recovery command is printed |
